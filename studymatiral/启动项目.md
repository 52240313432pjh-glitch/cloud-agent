好，现在按“本地 Qwen2.5 负责 LLM，DashScope 继续负责向量模型”的方式重启。

**1. 先启动本地 Qwen2.5**

如果你用 vLLM：

```bash
source .venv_vllm/bin/activate

vllm serve Qwen/Qwen2.5-7B-Instruct \
  --host 0.0.0.0 \
  --port 8001 \
  --dtype auto \
  --max-model-len 4096
```

如果你实际部署的是 3B 或 14B，把模型名换成你本地启动时的名字。

新开一个终端测试：

```bash
curl http://localhost:8001/v1/models
```

能返回模型列表就说明本地模型服务已启动。

**2. 修改项目的 agent/.env**

```bash
cd /mnt/d/项目实战/deep_research/cloud_agent
nano agent/.env
```

确保 LLM 指向本地 Qwen：

```env
BASE_URL=http://localhost:8001/v1
MODEL=Qwen/Qwen2.5-7B-Instruct
```

向量模型保持 DashScope，不要改：

```env
DASHSCOPE_API_KEY=你的阿里云百炼key
EMBEDDING_MODEL=text-embedding-v2
```

**3. 启动基础服务**

```bash
docker start redis
docker start mysql8
```

Neo4j：

```bash
docker start neo4j
```

如果 Milvus 是 docker compose：

```bash
cd /你的/milvus/部署目录
docker compose up -d
```

检查：

```bash
docker ps
```

**4. 启动后端**

新开终端：

```bash
cd /mnt/d/项目实战/deep_research/cloud_agent/app
source ../.venv/bin/activate
uvicorn app_main:app --host 0.0.0.0 --port 5000 --reload
```

看到类似：

```text
Application startup complete.
```

就可以了。

**5. 启动前端**

再开终端：

```bash
cd /mnt/d/项目实战/deep_research/cloud_agent/front/cloud_agent
npm run dev
```

打开：

```text
http://localhost:5173/
```

**6. 简单测试**

先测本地 Qwen：

```bash
curl http://localhost:8001/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "Qwen/Qwen2.5-7B-Instruct",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

再测项目：

```text
云服务器ECS有哪些基本属性？
```

如果后端日志里请求没有打到 DashScope 的 LLM，而是本地 `http://localhost:8001/v1`，就说明接入成功了。
